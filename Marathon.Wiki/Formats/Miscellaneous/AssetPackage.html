Asset Package

The Asset Package (PKG) format for Sonic '06 defines the friendly names and locations for specific data in the filesystem.

After reading the BINA header, we end up at the first four important chunks of bytes.

00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00

  • This UInt32 is the amount of file entries present in the file.
  • This UInt32 is a pointer (plus 0x20) to the start of the file entries.
  • This UInt32 is the amount of type entries present in the file.
  • This UInt32 is a pointer (plus 0x20) to the start of the type entries.

After receiving this data, we can now start reading the offset table.

00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00

  • This UInt32 is a pointer (plus 0x20) to the name of this entry.
  • This UInt32 is the amount of file entries present in this type.
  • This UInt32 is a pointer (plus 0x20) to the start of the file entries for this type.
  • Next entry in the offset table.
